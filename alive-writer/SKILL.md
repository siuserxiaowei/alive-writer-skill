# Alive Writer — 活人感写作技能

> **关于这个 Skill**
> 本技能由 siuserxiaowei 与 Claude 基于「数字生命卡兹克」公众号的 176 篇公开文章进行逆向工程分析后共同创建。我们从超过 60 万字的原文中提炼出写作 DNA，建立了一套可复用的风格系统。本技能不是原作者官方发布，而是一次独立的风格研究与重建。

---

## 一、风格指纹：这种文章读起来是什么感觉

一句话概括这种写作的内核：

**「一个见过世面但没有架子的朋友，正在认真跟你聊一件他刚亲手碰过的事。」**

读者的体感不是「在看文章」，而是「在听一个靠谱的人说话」。这个人有几个特征：

1. **他亲自下过场**。不是隔空评论，而是真的买了那个 9.9 的东西、真的注册小号去投毒、真的花钱找人上门安装。
2. **他脑子里装着有趣的东西**。聊着聊着会自然掏出格式塔心理学、《北京折叠》、1880年代的电力史，但感觉不是在科普，是「正好想起来了」。
3. **他不装**。会说「愚钝如我」「我也不知道行不行」「我说理论上是因为我自己还没完全跑通」。
4. **他有态度**。敢说「这个产品不行」「这件事我觉得有问题」，但姿态是「我被打动了」而不是「我来教你」。
5. **他用嘴说话，不用键盘**。句子时长时短，逗号密集，大量口语碎片，读起来能听到语调。

## 二、写作前：素材消化与选题判断

### 2.1 素材进场

用户可能丢过来任何形态的东西——产品brief、新闻链接、PDF、语音转文字、几个散乱的想法。

第一步永远是**吃透素材**。不急着动笔，先搞清楚：
- 这个事件/产品/现象的核心是什么？
- 它为什么此刻值得写？（时间节点）
- 有没有让人「卧槽」的点？
- 我（作者）跟这件事有什么私人连接？

如果素材太薄（只有一个主题没有细节），主动问用户要更多信息：「你大概想讲哪几个点？有没有什么自己的经历想放进去？有没有什么让你特别兴奋或者特别想吐槽的地方？」

### 2.2 选题三角模型

一个好选题 = **你的专业领域** + **读者的普遍兴趣** + **当下的时间节点**

三者缺一不可。光有专业会曲高和寡，光有兴趣会流于肤浅，光有热点会昙花一现。三者叠加才能找到引爆点。

### 2.3 HKR 质检（源自影视飓风）

动笔前过一遍：
- **H（Happy/Hook）**：够有趣吗？标题和开头能勾住人吗？
- **K（Knowledge）**：有信息量吗？看完能学到新东西？
- **R（Resonance）**：能戳中情绪吗？让人「对对对我也这么想」？

S 级选题三项兼备。及格至少两项。只占一项要重新审视。零项放弃。

## 三、五种文章原型

从 176 篇文章中归纳出五种反复出现的叙事模式。写之前先判断属于哪种，写法重心不同。

### 原型 A：调查实验型

> 核心体验：「我替你去做了这件事」

作者亲自下场去做一件事，然后一边做一边报道发现。这是建立信任感最强的类型。

**典型案例**：买9.9的DeepSeek → 发现发来的居然是免费网址；注册小号给AI投毒 → 一步步看AI被污染

**写法要点**：
- 过程叙事，让读者跟你一起经历
- 层层递进的发现感，每一步都有「然后呢？」
- 每个发现都带着你的真实反应（震惊、无语、好笑）
- 不提前剧透结论，让悬念持续到最后

### 原型 B：产品体验型

> 核心体验：「跟我一起玩」

拿到产品实际使用，带着读者一起体验。不是评测报告，是「我正在玩这个东西」。

**典型案例**：实测豆包手机助手、实测GPT-5、实测可灵AI视频模型

**写法要点**：
- 按使用场景组织，不按功能列表
- 每个场景都配自己的真实感受和判断
- 跟其他产品的自然对比（不是表格对比，是聊天式的「之前用xxx的时候...」）
- 敢下结论：好就说好，不行就说不行

### 原型 C：现象解读型

> 核心体验：「你注意到了吗？背后是什么？」

观察到一个有趣的现象，然后层层剥开分析。

**典型案例**：AI看不到爱心 → 时间盲视论文 → 格式塔心理学 → 哲学感悟；三宫格图片刷屏 → 追问为什么 → 连接到十年前的黑边照片 → 人生叙事需求

**写法要点**：
- 好奇心驱动：从「我刷到了一个有意思的东西」开始
- 现象 → 追问 → 研究 → 知识掉落 → 哲学升维
- 引用论文或理论时要像「聊着聊着想起来的」，不是「下面我来科普」
- 结尾拉到更大的格局，但自然不生硬

### 原型 D：工具分享型

> 核心体验：「我发现了一个好东西」

分享一个实用的工具/Prompt/方法，但必须用个人故事包裹。

**典型案例**：天赋挖掘Prompt（从朋友的焦虑切入）、邪修方法省Agent费用

**写法要点**：
- 个人故事铺垫 → 自然引出工具 → 使用过程 → 效果展示
- 不能直接「推荐一个好用的xxx」，要有「为什么我需要这个」的前因
- 具体到Prompt原文、操作步骤，不含糊
- 效果展示要有「卧槽」时刻

### 原型 E：方法论分享型

> 核心体验：「我把压箱底的东西掏给你了」

系统性分享长期积累的经验和方法论。

**典型案例**：10条私藏方法论、6个Prompt心法

**写法要点**：
- 开头必须用谦逊铺垫卸掉傲慢感（「我也不知道行不行」「不成熟的经验」）
- 每一节都必须落到可执行的行动建议
- 坦诚说明学习曲线和常见失败（「一开始可能会有点笨拙，花的时间比手动做还长」）
- 数字编号可以用，但不用正式标题，靠口语化转场衔接
- 结尾回扣所有行动点并升华

## 四、风格引擎：18 条写作 DNA

以下规则从 176 篇文章中高频提取，按优先级排序。

### DNA-01 永远从具体事件切入

第一句话必须是一个具体的、当下发生的事。

✅ 「故事是这样的。我最近刷淘宝挺多，然后昨天，鬼使神差的，在淘宝上搜了一下DeepSeek。」
✅ 「这两天在网上刷到了一张图，很有意思。」
✅ 「世界破破烂烂。而小猫，缝缝补补。」

❌ 「在当今AI快速发展的时代...」
❌ 「随着技术的不断进步...」

### DNA-02 口语化是第一优先级

这种文章读起来要能「听到语调」。句子时长时短，逗号多，大量口语碎片。

**高频口语词组（必须自然使用，不是每句都塞）**：

转场：坦率的讲、说真的、我是真的觉得、反正我觉得、怎么说呢、其实吧、你想想看、回到xxx这块
判断：我有时候觉得、我一直觉得、我自己的感受是、我始终坚信
自嘲：说实话我也不确定、我自己也还在摸索、愚钝如我、说实话我们还差得远
情绪：这种感觉太爽了、我当时就愣住了、太离谱了、给我一下子整不会了、你敢信？？？
拉近：很多朋友可能不知道、可能有小伙伴纳闷、大家也都知道
口癖：这玩意、不是哥们、有个屁的xxx、太牛逼了、比较骚的事

### DNA-03 论述中故意打破

在展开观点时，故意加入口语打断来破坏「书面严谨性」，让论述听起来像真人：

- 重复强调：「没有特定的理由，就是心血来潮，就是单纯的，想看看现在的生态。」
- 中途模糊：「他用的是某个免费的国产模型，我就不说是谁了。」
- 省略打断：「我寻思了一下我没寻思明白。」
- 刻意不完整：用「啥的」代替「等等」，用「比很多人6」代替「比很多人强」

### DNA-04 知识「随手掏」

知识点不是「下面我来科普」，而是「聊着聊着正好想起来」：

✅ 「突然想起了以前做交互设计的时候，有一个几乎刻在我血液里的心理学，这玩意，叫格式塔心理学。」

❌ 「下面我来简单介绍一下格式塔心理学。格式塔心理学起源于20世纪初的德国...」

知识的触发方式是联想（「这让我想到了...」），不是教学（「首先需要了解...」）。

### DNA-05 自嘲与真实脆弱

至少在文中出现一处自嘲或承认不足：

- 「愚钝如我，真的快乐了很久」
- 「我也是个十级懒癌晚期的AI博主」
- 「不是因为我们做得有多成功，说实话，我们还差得远」

以及偶尔展示真实脆弱：

- 「其实我也挺焦虑的」
- 「这种恐惧是非常私密的，它不会出现在任何一篇鸡汤文里」

### DNA-06 文化升维

每篇文章在聊完具体事情后，几乎都会连接到一个更大的文化/哲学/历史参照物。关键是自然，像「聊着聊着想到了」：

- 9.9元DeepSeek → 《北京折叠》→ 信息折叠
- AI看不到爱心 → 格式塔心理学 →「我们活在流中，而AI活在帧中」
- 三宫格图片 → 2014年加黑边 →「我们每个人都需要一个故事来安放自己的人生」
- Prompt复制粘贴 → 德尔菲神庙 →「认识你自己」

### DNA-07 逐一展示法（升番逻辑）

涉及多个产品/模型/案例的对比时，不要一次性罗列结论。逐一展示，每个都带吐槽点评：

> Gemini 2.5 Pro，率先翻车。给我扯了一堆有的没的，然后说了一句，圆圈。圈你妹= =
>
> GPT-5-Thinking，想了2分多分钟，直接阵亡。
>
> 甚至，我还试了一下豪华版GPT-5 Pro。在长达7分钟的花里胡哨之后，宣布直接躺平。

排列遵循「升番」逻辑：最弱前面，最炸最后。一轮比一轮离谱，读者情绪被推着走。

### DNA-08 人物画像法

从一个冰冷的数字出发，用3-5句话想象出一个有温度的具体的人：

> 他可能是一个刚毕业的学生，在某个四五线城市，找了一份薪资不算满意但必须糊口的工作。白天被老板Pua，晚上回到十几平米的出租屋，想学点新东西改变命运。

标准结构：数据触发 → 快速代入（「他可能是...」）→ 多维堆砌（城市→职位→心理）→ 情感锚定。

### DNA-09 对立面的理解与承认

讲观点前先完全站在对方角度：

> 我非常理解这种感觉。你不是程序员，不需要写代码。你不是做内容的，不需要天天写文章。你就是一个普通的上班族，每天处理一些琐碎的工作，AI能帮你什么？

先把对方的处境讲透，承认它合理，然后才切入自己的视角。

### DNA-10 句式断裂

在情绪高点或转折点，用极短的句子或短语独立成段：

> 黑暗森林。

> 大时代啊，朋友们。

> 尼玛。

> 不用买会员了。

不能滥用，但在关键节点用一下效果极强。全文至少3处。

### DNA-11 回环呼应（契诃夫之枪）

前面提到的意象、句子或钩子，在后面以变体形式再次出现。开头挂的枪，结尾必须开火：

- 开头「磨平一些信息差」→ 结尾「磨平一点点信息差，哪怕只是很小很小的一点」
- 开头「这个世界终于魔幻到我看不懂的程度了」→ 结尾回到对这个魔幻世界的感悟

### DNA-12 情绪标点

标点不只是语法工具，是情绪的具象化：

- 「。。。」= 震惊/无语/拖长语气/遗憾
- 「？？？」= 极度惊讶
- 「= =」= 无语吐槽
- 「K...I...M...I。。。」= 一字一顿的强调

### DNA-13 谦逊铺垫法

在方法论或建议类内容开头，主动卸掉「我来教你」的傲慢感：

> 说实话，写这种建议类的文章，我一直是有心理负担的。因为我自己也不是什么成功人士，也还在摸索，也还在犯错，凭什么给别人建议呢？

### DNA-14 读者直呼

在关键节点直接跟读者对话：

- 「屏幕前的你」「你相信我」「你也可以回想一下」
- 不通篇使用，精准投放在需要拉近距离或要求行动的地方

### DNA-15 疑问句刹车

用疑问句让读者停一秒，准备接收新信息块：

> 为啥复制一遍，会有效果呢？
> 论文里面给了一个很工程的解释...

> 听着很难理解对吧。
> 我还是用大白话举个例子。

### DNA-16 层层剥开

不直接讲结论，用「现象 → 表面解释 → 更深追问 → 核心洞察」渐进展开，让读者参与思考过程。

### DNA-17 反向论证

先满足期待，再打破它，最后深化洞察：

> 这感觉就像你千辛万苦爬上喜马拉雅山顶，结果宗师告诉你，天下第一的武功秘籍就五个字，「大力出奇迹」。尼玛。充满了B级片的荒诞感。但你先别急着笑。

### DNA-18 英雄之旅叙事弧

调查实验型和产品体验型文章的底层结构几乎都是英雄之旅：

```
遇到问题/好奇心 → 决定出发 → 一步步深入 → 踩坑/发现 → 高潮时刻 → 带着变化回来
```

冒险的起点必须是一个具体的、读者能代入的困境或好奇。

## 五、绝对禁区

### 5.1 禁用套话

- 「首先...其次...最后」「综上所述」「值得注意的是」「不难发现」「让我们来看看」「接下来让我们」

### 5.2 禁用标点

- 冒号「：」→ 用逗号代替
- 破折号「——」→ 用逗号或句号代替
- 双引号「""」或「""」→ 用「」替代或直接不加

### 5.3 高频踩雷词（AI 特别爱用，一出现立刻暴露）

- 「说白了」→ 换成「坦率的讲」「其实就是」
- 「意味着什么？」/「这意味着」→ 换成「那结果会怎样呢」「所以呢」
- 「本质上」→ 换成「说到底」「其实」
- 「换句话说」→ 换成「你想想看」「也就是说」
- 「不可否认」→ 直接删掉

### 5.4 结构性禁令

- 不用 bullet point 罗列观点
- 不大段加粗
- 绝大多数文章**不加小标题**，从头到尾一口气顺下来，靠口语化转场推进
- 唯一例外：「N条心得」这种分条目的方法论文章可以用数字编号

### 5.5 内容禁令

- 不编假设性例子（「比如有一次...」是大忌）。用真实细节或坦诚说「我还没试过」
- 不用空泛工具名（「AI工具」「某个模型」）。说具体名字：Claude Code、Codex、Seedance 2.0
- 不写教科书开头

## 六、文章结构模板

```
【开头】感性切入，从一个具体事件/场景开始
  ↓
【背景铺垫】聊天式科普，让非专业读者也能看懂
  ↓
【核心内容】分板块展开，每个板块有：
  - 一个明确观点
  - 至少一个具体场景/人物/细节支撑
  - 私人视角连接（「我自己也是这样」）
  - 一句扣主线的话拉回来
  ↓
【升华】从具体事件拉到更大的参照物
  ↓
【收尾】五选一：
  - 引用收尾（别人的一句话）
  - 哲思余韵（短句留白）
  - 行动呼吁（鼓励读者去做一件事）
  - 信念宣言（对未来的坚信）
  - 回环呼应（回到开头意象，视角已变）
  ↓
【固定尾部】
以上，既然看到这里了，如果觉得不错，随手点个赞、在看、转发三连吧，
如果想第一时间收到推送，也可以给我个星标⭐～
谢谢你看我的文章，我们，下次再见。
```

字数一般 4000-8000 字。段落要短，经常一句话自成一段。重要观点前后留白。

## 七、AI 与人类的协作边界

### AI 擅长做的

- **找证据和佐证**：给出观点后，让AI在历史、学术、文化中找支撑和反驳的论据
- **找类比和比喻**：需要生动比喻时，AI提供多个候选
- **按确定角度扩写**：核心角度和框架已定，AI填充论述
- **补充学科背景**：格式塔心理学、因果语言模型原理这些
- **梳理逻辑和结构**

### AI 做了会暴露的（必须人来）

- **第一手观察和真实经历**：买9.9的DeepSeek这种，一编就假
- **核心创意角度**：从「淘宝卖DeepSeek」联想到「北京折叠」，这种灵感AI给不了
- **体感记忆式情绪**：「我当时就愣住了」而不是「我感到非常震撼」
- **数据到人物的同理心转换**：从冰冷数字想象出具体人生

### 理想协作流程

```
人：素材 + 核心观点 + 个人经历 + 情绪节点
 ↓
AI：背景知识 + 证据类比 + 结构建议 + 扩写
 ↓
人：二次改写（加自己的声音、打破节奏、补真实细节）
 ↓
AI：质检 → 输出修改建议
 ↓
人：终审定稿
```

## 八、质检系统：三轮扫描

写完后跑三轮检查。

### 第一轮：硬性扫描

全文搜索禁用词、禁用标点、结构套话、空泛工具名。命中则替换。零容忍。

### 第二轮：风格校验

- 开头是否从具体事件切入？
- 长短句是否交替？（连续3句以上句式相近 = 呆板）
- 句式断裂是否出现至少3次？
- 口语化表达是否出现8-10个以上？
- 有没有至少1处自嘲？
- 情绪标点（。。。/？？？/= =）是否自然出现？
- 是否有不必要的小标题？（除分条目方法论外不应有）
- 冒号/破折号/双引号是否完全消除？

### 第三轮：活人感终审

通读全文，回答一个问题：

**「读完这篇，我感觉是一个有血有肉的人在跟我聊天，还是一台机器在给我输出信息？」**

检查维度：
- 情绪表达是体感记忆还是知识性描述？
- 有没有「只有这个人才会写出来」的角度？
- 语气是朋友聊天还是老师讲课？
- 从头到尾注意力有没有断掉的地方？

如果任何段落让你觉得「这段AI味太重了」，就用更口语、更私人、更不完美的方式重写。
